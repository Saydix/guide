### Режим работы  
<img width="995" alt="image" src="https://github.com/user-attachments/assets/3f57d4c4-d27c-41cd-91f7-1f81ce85b0bc" />  
Доступно 3 режима работы:</br>
1. Rule (Набор настраиваемых правил) (Используется в 92% случаев)  
2. Global (Весь трафик идет через прокси)  
3. Direct (Весь трафик идет прямо, без прокси)

## Опции редактирования профиля  
<img width="741" alt="image" src="https://github.com/user-attachments/assets/ece92522-f6cc-4488-bffe-e2d882ba5060" />   

В случае необходимости можно добавлять и удалять правила     
Наиболее удобный инструмент - "редактировать правила"    
### Редактировать правила  
<img width="900" height="800" alt="image" src="https://github.com/user-attachments/assets/00355f79-3276-4941-864d-ee3d836a1532" />   

Тип правил: (Простые в использовании закрывающие 98% потребностей)    
1. DOMAIN-SUFFIX. Ищет правила по домену второго (wikipedia) и верхнего (.org) уровней    
2. GEOSITE. Получает список сформированных сообществом ссылок на определённый ресурсы (Позволяет не вписывать десятки доменов необходимых для работы сайта вручную. Например: youtu.be, yt.bem googlevideo.com youtubekids.com youtubei.googleapis.com)   
3. PROCESS-NAME. Позволяет проксировать трафик приложений. Необходимо чтобы не добавлять десятки доменов и ip адресов, т.к. некоторые приложения подключаются напрямую, как например это делает телеграмм  

Содержимое правила: Домен, ip, название процесса  

Политика прокси:  
1. Direct. Трафик идет напрямую  
2. Proxy. Трафик идет через прокси  
3. Reject/Reject-Drop. Отменяет запросы/Уничтожает их. Необходимо для отключения тилеметрии и рекламы, уменьшения нагрузки на трафик и прокси  
4. Match. Весь оставшийся трафик, который не обработало ни одно из правил  
Добавить правило в начало и добавить правило в конец  
Чаще всего лучше добавлять правила в начало, т.к. обращение с правилами работает сверху вниз. То есть правила сверху будут использоваться в первую очередь, а если вам необходимо пере-определение каких-либо инструкций лучше установить ваше правило в начало  

## Изменить файл
<img width="900" height="800" alt="image" src="https://github.com/user-attachments/assets/7d4bd13c-b9c5-40e3-a991-ccd646950c5f" />   
Необходимо для ручного редактирования и добавления правил. По сути тоже самое что и Редактировать правила, только без интерфейса и с большим контролем   
Однако сохранённые изменения в файл сохраняются до его обновления на сервере. То есть для поддержания актуального соединения и конфигураций лучше использовать [Редактировать правила](#редактировать-правила), иначе придётся отказаться от обновлений конфигурации поставщиком вашего прокси   
Раздел rule описывает те же настройки что и [Редактировать правила](#редактировать-правила)   

# Вкладка Соединения
<img width="1500" height="600" alt="image" src="https://github.com/user-attachments/assets/e6808596-a1fb-4958-8400-6c5533a1fc70" />  
Основная вкладка для анализа трафика и выявления доменов, процессов и адресов, которые используются приложениями. Может показаться перегруженной, но в ней есть удобне поле фильтрации и вчитавшись будет не сложно нужную информацию  
Столбцы:   
Хост. Показывает ip адрес или домен к которому идёт запрос   
Цепочки. Показывает как именно идет соединение к запрашиваемому ресурсу. Через Proxy или DIRECT напрямую </br>
Правило. Показывает правило, которое было применено к соединению   
Процесс. Показывает процесс из которого был вызван запрос     
Т.к. мой конфиг пользуется концепций черных списков, а не белых - большинство соединений будет идти через Match, то есть прямо, без прокси. И только указанные в правилах ресурсы будет проксировать   
Если в [режим работы](#режим-работы) выбрать Global то можно заметить как весь трафик будет идти через прокси   

# Вкладка Настройки  
<img width="1500" height="600" alt="image" src="https://github.com/user-attachments/assets/3a09deb3-2999-462c-93c4-6c7595b35c9b" />  
Тут пройдемся только по основным   
Автозапуск: Позволяет программе автоматически запускаться с системой. Удобно если настроить все правила. Можно забыть о включении/отключении впн и просто наслаждаться жизнью   
Тихий запуск: Окно вообще не будет всплывать при запуске системы   

IPv6: Пока что стоит отключить, т.к. прокси не настроен на работу с ним   

Язык: Лучше выбрать английский, не все пункты переведены адекватно   

Настройки сочетаний клавиш: Можно настроить включение отключение по горячим клавишам   

Цветовая тема: Можно сменить на приятный глазу цвет   

# FAQ
### Нужный мне сервис работает не через прокси   
- Сервис работает в браузере  
В первую очередь необходимо взять его адрес и добавить через [Редактирование правил](#редактировать-правила)  
На примере ссылок:   
https://saydix.github.io/guide/clash-verge-guide, необходимо взять только github.io  
https://ru.wikipedia.org/wiki/%D0%94%D1%80%D0%B5%D0%B7,_%D0%9A%D0%B0%D1%80%D0%BB, необходимо взять только wikipedia.org  
В случае если сервис заработал, но не полностью (Например не грузит картинки, но при этом хорошо грузит текст) значит сервис использует несколько доменов, в таком случае необходимо добавить его через набор GEOSITE, который доступен по ссылке https://github.com/v2fly/domain-list-community/tree/master/data  
Чаще всего набор называется в честь доменного имени сайта. Например:  
https://saydix.github.io/guide/clash-verge-guide - github  
https://www.youtube.com/watch?v=dQw4w9WgXcQ - youtube  
- Сервис работает через приложение  
Через [Редактирование правил](#редактировать-правила) необходимо добавить название процесса  
<img width="1200" height="200" alt="image" src="https://github.com/user-attachments/assets/b9421202-bf04-45dc-88b2-c419af3bcb0f" />  
Стоит убедиться что все процессы  идут через прокси, некоторые приложения могут использовать несколько процессов  
<img width="1200" height="150" alt="image" src="https://github.com/user-attachments/assets/e31e0065-c414-4ba6-b8c7-f8265334e39f" />  


### Нужный мне сервис работает через прокси, а должен напрямую
Если сервис не заблокирован, но при этом по какой-то причине запускается через прокси в первую очередь следует проверить [режим работы](#режим-работы). Он должен быть установлен на Rule
Если сервис всё ещё идет через прокси стоит добавить его в начало списка через [Редактирование правил](#редактировать-правила) с DIRECT политикой прокси
