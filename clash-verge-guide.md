### Режим работы  
<img width="995" height="266" alt="image" src="https://github.com/user-attachments/assets/3f57d4c4-d27c-41cd-91f7-1f81ce85b0bc" />  
Доступно 3 режима работы:</br>
1. Rule (Набор настраиваемых правил) (Используется в 92% случаев)</br>
2. Global (Весь трафик идет через прокси)</br>
3. Direct (Весь трафик идет прямо, без прокси)</br>

## Опции редактирования профиля  
<img width="741" height="496" alt="image" src="https://github.com/user-attachments/assets/ece92522-f6cc-4488-bffe-e2d882ba5060" /> </br>
В случае необходимости можно добавлять и удалять правила</br>  
Наиболее удобный инструмент - "редактировать правила"</br>  
### Редактировать правила
<img width="1534" height="1161" alt="image" src="https://github.com/user-attachments/assets/00355f79-3276-4941-864d-ee3d836a1532" />
Тип правил: (Простые в использовании закрывающие 98% потребностей)</br>
1. DOMAIN-SUFFIX. Ищет правила по домену второго (wikipedia) и верхнего (.org) уровней</br>
2. GEOSITE. Получает список сформированных сообществом ссылок на определённый ресурсы (Позволяет не вписывать десятки доменов необходимых для работы сайта вручную. Например: youtu.be, yt.bem googlevideo.com youtubekids.com youtubei.googleapis.com)</br>
3. PROCESS-NAME. Позволяет проксировать трафик приложений. Необходимо чтобы не добавлять десятки доменов и ip адресов, т.к. некоторые приложения подключаются напрямую, как например это делает телеграмм</br>
Содержимое правила: Домен, ip, название процесса
Политика прокси: 
1. Direct. Трафик идет напрямую
2. Proxy. Трафик идет через прокси
3. Reject/Reject-Drop. Отменяет запросы/Уничтожает их. Необходимо для отключения тилеметрии и рекламы, уменьшения нагрузки на трафик и прокси
4. Match. Весь оставшийся трафик, который не обработало ни одно из правил
Добавить правило в начало и добавить правило в конец
Чаще всего лучше добавлять правила в начало, т.к. обращение с правилами работает сверху вниз. То есть правила сверху будут использоваться в первую очередь, а если вам необходимо пере-определение каких-либо инструкций лучше установить ваше правило в начало

## Изменить файл
<img width="1533" height="1163" alt="image" src="https://github.com/user-attachments/assets/7d4bd13c-b9c5-40e3-a991-ccd646950c5f" />   
Необходимо для ручного редактирования и добавления правил. По сути тоже самое что и Редактировать правила, только без интерфейса и с большим контролем </br>
Однако сохранённые изменения в файл сохраняются до его обновления на сервере. То есть для поддержания актуального соединения и конфигураций лучше использовать [Редактировать правила](#редактировать-правила), иначе придётся отказаться от обновлений конфигурации поставщиком вашего прокси </br>
Раздел rule описывает те же настройки что и [Редактировать правила](#редактировать-правила) </br>

# Вкладка Соединения
<img width="2212" height="1254" alt="image" src="https://github.com/user-attachments/assets/e6808596-a1fb-4958-8400-6c5533a1fc70" />  
Основная вкладка для анализа трафика и выявления доменов, процессов и адресов, которые используются приложениями. Может показаться перегруженной, но в ней есть удобне поле фильтрации и вчитавшись будет не сложно нужную информацию  
Столбцы: </br>
Хост. Показывает ip адрес или домен к которому идёт запрос </br>
Цепочки. Показывает как именно идет соединение к запрашиваемому ресурсу. Через Proxy или DIRECT напрямую </br>
Правило. Показывает правило, которое было применено к соединению </br>
Процесс. Показывает процесс из которого был вызван запрос </br>  
Т.к. мой конфиг пользуется концепций черных списков, а не белых - большинство соединений будет идти через Match, то есть прямо, без прокси. И только указанные в правилах ресурсы будет проксировать </br>
Если в [режим работы](#режим-работы) выбрать Global то можно заметить как весь трафик будет идти через прокси </br>

# Вкладка Настройки  
<img width="2218" height="1253" alt="image" src="https://github.com/user-attachments/assets/3a09deb3-2999-462c-93c4-6c7595b35c9b" />  
Тут пройдемся только по основным   
Автозапуск: Позволяет программе автоматически запускаться с системой. Удобно если настроить все правила. Можно забыть о включении/отключении впн и просто наслаждаться жизнью   
Тихий запуск: Окно вообще не будет всплывать при запуске системы   

IPv6: Пока что стоит отключить, т.к. прокси не настроен на работу с ним   

Язык: Лучше выбрать английский, не все пункты переведены адекватно   

Настройки сочетаний клавиш: Можно настроить включение отключение по горячим клавишам   

Цветовая тема: Можно сменить на приятный глазу цвет
